# Contributing

Thank you for contributing! By contributing to this project you agree that your
contributions will be licensed under the terms of the repository `LICENSE` file.

Please follow these guidelines when opening issues or pull requests:

- Create a feature branch using the pattern `feature/short-description` or
  `fix/short-description`.
- Keep PRs small and focused.
- Add a clear PR description explaining the change and why it's needed.
- Ensure commits include a `Signed-off-by: Your Name <you@example.com>` line
  (Developer Certificate of Origin). You can add this automatically with:

```
git commit -s -m "Your commit message"
```

- Ensure any third-party code you add is compatible with the repository license.
- Run any tests and linters before opening a PR.

If you have questions, open an issue and we'll help shape the change.
